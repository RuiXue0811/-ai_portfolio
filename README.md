# 金融用户复贷触达 Bot

> 基于 Coze 搭建的 AI 工作流项目。把「知道这个用户是谁」到「给他发什么、用什么渠道发」的全链路自动化——用户分层、合规话术生成、触达策略建议，一次跑通。

## 核心功能

### 7 节点全自动化工作流

| 节点 | 功能 | 关键设计 |
|------|------|---------|
| 用户分层 | A/B/C/D 四层识别 | 命中即停判定顺序，准确率 0%→100% |
| 知识库检索 | RAG 检索合规话术库 | 按用户层级匹配最优话术模板 |
| 话术生成 | 3 种风格输出 | 正式 / 关怀 / 简洁，适配不同渠道 |
| 合规审查（初筛） | 7 类红线词过滤 | 关键词黑名单硬过滤 |
| 合规审查（深审） | LLM 语义级合规判定 | 4 个 PASS/FAIL 对照示例，通过率 0%→89% |
| 触达策略 | 渠道 / 时间 / 紧急度建议 | 结合用户层级和历史行为综合推荐 |
| 输出整合 | 结构化 JSON 输出 | 方便下游系统直接消费 |

### 用户分层设计

| 层级 | 特征 | 策略方向 |
|------|------|---------|
| A 层 | 结清 ≤30 天，高信用 | 主动邀约，高优先级 |
| B 层 | 结清 31~90 天，良好记录 | 常规触达，利率优惠切入 |
| C 层 | 结清 91~180 天，沉默用户 | 唤醒策略，低门槛产品 |
| D 层 | 逾期记录或高风险 | 暂缓触达，合规优先 |

## 关键 Bug 与解法

**B1 — 合规节点全军覆没**
- 根因：prompt 只列规则，LLM 把 warning 当 fail
- 修复：加 4 个 PASS/FAIL 示例 + 显式二值化判定
- 教训：示例往往比规则文字更有效

**B2 — 变量名模板字符串泄露**
- 根因：变量名拼错（claer→clear），Coze 严格匹配，拼错静默降级为字面量
- 修复：改名 + 改从「输入」区接字段

**B3 — 用户层级判断错误**
- 根因：LLM 数值比较盲区（days=75>30 判成了 A 层）
- 修复：加「D→A→B→C→兜底，命中即停」+ 把反例写进 prompt

## 技术架构

```
用户数据输入（结清天数 / 信用评分 / 历史行为）
        ↓
Coze 工作流（7 节点）
        ↓
LLM（Prompt Engineering + RAG 知识库检索）
        ↓
合规审查（关键词过滤 + 语义级 LLM 审查双重保障）
        ↓
结构化输出（话术 + 渠道 + 触达建议）
```

- **平台**：Coze 无代码工作流，降低维护门槛
- **核心技术**：Prompt Engineering、RAG 知识库检索、代码节点
- **合规保障**：双层审查（硬过滤 + LLM 语义审查）

## 项目背景

这是我在天创信用服务担任策略运营期间，基于 3 年真实信贷运营经验独立完成的 side project。核心挑战：

- **业务建模**：把"分层靠人工、话术靠感觉、合规靠运气"的 SOP 拆解成可被 AI 执行的节点逻辑
- **Prompt 工程**：用示例对比、命中即停、硬规则前置等技巧解决 LLM 在金融合规场景下的边界问题
- **可复用性**：沉淀分层规则和合规话术库后，换一批用户数据直接跑
