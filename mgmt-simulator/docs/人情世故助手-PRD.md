# 人情世故助手 PRD

## 一、产品概述

### 1.1 产品定位
一款基于AI对话的人情世故学习与实战工具，帮助用户识别复杂人际局面中的潜在风险、解读他人动机，并提供防身应对策略。

### 1.2 目标用户
- **职场新人/中层**：面对复杂的办公室政治，容易被试探、被利用
- **社交敏感人群**：担心"被人害但看不出来"，希望提升识局能力
- **跨文化/代际沟通者**：亲戚、家族聚会中的人情往来困扰
- **创业者/管理者**：需要在利益博弈中保护自己

### 1.3 核心价值主张
> "别人已经在博弈了，而你还没意识到局已经开始"

我们提供：
- **识局能力**：看出试探、苗头、潜台词
- **读人框架**：判断动机和立场
- **防身策略**：不撕破脸地保护自己
- **应对话术**：具体行动建议

## 二、核心功能

### 2.1 每日情局卡（主动学习）

#### 2.1.1 卡牌体系

**四大卡组**：

| 卡组名称 | 核心能力 | 卡片数量 |
|---------|---------|---------|
| **识局卡** | 发现试探和苗头 | 30+ |
| **读人卡** | 判断动机和立场 | 30+ |
| **防身卡** | 不撕破脸地保护自己 | 30+ |
| **应对卡** | 具体行动和话术 | 30+ |

#### 2.1.2 抽卡机制

**触发方式**：
- 每日首次打开自动抽取
- 用户可主动选择卡组抽取
- 完成一张卡后可连续抽下一张（不限制）

**抽卡规则**：
- 算法优先推荐用户未学习过的卡片
- 用户可标记"想再看一次"的卡片加入复习池
- 已完成的卡片可在卡册中回顾

#### 2.1.3 卡片交互流程

```
1. 呈现场景（150-300字）
   ↓
   来自《晚明》改编的真实情局描述
   
2. 苏格拉底式提问（AI主动）
   ↓
   "这个局面里，你觉得哪里不对劲？"
   "如果是你，你会怎么判断X的真实意图？"
   
3. 用户作答（自由输入）
   ↓
   
4. AI推演（军师式）
   ↓
   "如果你这么做，可能会发生..."
   列出2-3种后果路径
   
5. 揭示真相
   ↓
   展示原著主角的实际应对
   
6. 提炼规律
   ↓
   "这个场景告诉我们：XXX"
   
7. 延伸思考（可选）
   ↓
   "你有遇到过类似的局吗？说说看"
   → 如果用户分享，进入【直接问诊模式】
```

#### 2.1.4 卡片示例

**【识局卡-001】陈新识破利益转移术**

**场景描述**：
> 你的下属周世发在城里搞"没收赃物"行动时，被发现私自掠夺百姓财产。你的盟友刘民有当众质问你，眼神里充满怀疑。此时你手上刚缴获了一大笔军资——价值百万两白银和千头牲畜。

**AI提问**：
- 这个局面里，你最大的风险是什么？
- 如果你直接解释"周世发不是我授意的"，会发生什么？

**用户作答区**

**AI推演**：
> 如果你选择辩解：
> - 刘民有会追问细节，怀疑会加深
> - 其他盟友也会开始怀疑你的管理能力
> - 周世发会觉得你不护短，产生离心
>
> 如果你选择严惩周世发：
> - 表面上问题解决，但刘民有已经看到了裂缝
> - 以后每次类似事件都会被放大镜检视

**揭示真相**：
> 陈新的做法：立即把刘民有的注意力转向城外的巨额缴获，详细计算这笔财富能解决多少人的吃饭问题。刘民有看到实际利益后，彻底忘记了对情报局的怀疑。

**提炼规律**：
> **当被识破小动作时，不要辩解，而是用更大的利益转移对方视线。人的贪欲会自动过滤掉不利信息。**

**延伸思考**：
你有遇到过"被质疑，但辩解会越描越黑"的局面吗？

### 2.2 直接问诊（被动响应）

#### 2.2.1 输入方式

**两种输入模式**：

**模式A：自由叙述**
```
用户直接描述："我跟你说件事，今天领导突然..."
```

**模式B：结构化填写**
```
【场景类型】□职场 □家庭 □社交 □其他
【关键角色】
【发生了什么】
【你的顾虑】
【你想要的结果】
```

系统根据用户输入复杂度，智能判断是否需要补充追问结构化信息。

#### 2.2.2 分析输出框架

**第一步：识局（苏格拉底式引导）**

AI不直接给答案，先反问：
- "这个人为什么要在这个时间点做这件事？"
- "你觉得TA真正想要的是什么？"
- "如果TA只是试探，TA想试探出你什么？"

**第二步：读人（军师式分析）**

呈现AI的综合判断：
```
【局势判断】
这个局面的本质是：XXX

【对方动机推演】
可能性1（60%）：XXX
可能性2（30%）：XXX
可能性3（10%）：XXX

【潜在风险】
如果不处理：XXX
如果处理失当：XXX
```

**第三步：防身+应对（方案推演）**

给出2-3个应对方案，每个方案包含：
- 具体动作/话术
- 预期效果
- 可能风险
- 适用条件

示例：
```
【方案A：沉默观察】
▸ 具体做法：对对方的试探不做明确回应，用"我再想想"拖延，观察TA接下来的动作
▸ 预期效果：如果对方继续施压，说明TA确实有所图；如果对方转向其他人，说明只是广撒网
▸ 风险：可能被认为"优柔寡断"，丧失先机
▸ 适用条件：你暂时没有明确立场，或者需要更多信息

【方案B：借力打力】
▸ 具体做法：把这个问题"转介"给更高层决策者，表面上是请示，实际上是让对方直接面对权力源
▸ 预期效果：对方如果只是试探你，不敢真的捅到上面，会自己退缩
▸ 风险：如果对方真有上面的支持，你会被动
▸ 适用条件：你怀疑对方在虚张声势

【方案C：主动摊牌】（不推荐）
▸ 具体做法：直接问对方"你是不是想XXX"
▸ 风险：撕破脸，没有回旋余地
▸ 适用条件：你已经掌握对方把柄，且准备好承担对抗后果
```

**第四步：互动推演**

用户可以说"我想试试方案A"，AI模拟对方可能的反应，继续推演下一步。

### 2.3 用户成长体系

#### 2.3.1 能力图谱

用户的学习行为会积累数据，生成个人能力雷达图：

```
      识局能力
          /\
         /  \
应对能力 ---- 读人能力
         \  /
          \/
      防身能力
```

每个维度根据以下数据计算：
- 完成的卡片数量
- 问诊中的准确判断次数
- 推演选择的成功率（用户反馈）

#### 2.3.2 卡册系统

用户可以：
- 查看已完成的卡片
- 标记"想再看一次"加入复习
- 查看相似情局的关联卡片
- 导出笔记（自己的作答+AI分析）

#### 2.3.3 真实案例库

用户可选择性地将自己的问诊记录（脱敏）贡献到社区案例库，其他用户可以：
- 看到这个真实局面
- 给出自己的判断
- 对比原主角的实际做法和结果

## 六、核心竞争力

### 6.1 vs. 传统书籍/课程
- **互动性**：不是单向输入，而是苏格拉底式引导+推演
- **即时性**：直接针对用户当下的真实困境

### 6.2 vs. 通用AI助手
- **垂直领域**：专注人情世故，有结构化的分析框架
- **真实素材**：基于《晚明》等经典文学作品，不是泛泛而谈

### 6.3 vs. 心理咨询
- **低门槛**：不需要预约，随时可用
- **场景化**：不是情绪疏导，而是具体局势分析

## 九、风险与应对

### 9.1 内容风险
**风险**：《晚明》场景过于古代化，用户代入感差
**应对**：
- 改编时转换为现代职场/社交语境
- 保留核心逻辑，替换具体场景元素

### 9.2 AI风险
**风险**：AI分析不准确，给出错误建议
**应对**：
- 在敏感场景添加免责声明
- 强调"仅供参考，最终决策需结合实际"
- 收集用户反馈，持续优化prompt

### 9.3 伦理风险
**风险**：产品可能被用于"害人"而非"防身"
**应对**：
- 在产品理念中强调"防守而非进攻"
- 对明显的恶意使用场景进行内容审核
- 建立用户行为准则
